HOW THE FIX WORKS:
------------------
BEFORE:  Two concurrent requests both see "no appointments at 10:00" 
         → Both insert → 2 appointments at same time ❌

AFTER:   First request inserts
         → Second request gets "serialization_failure" error
         → User gets 409 response: "Slot just booked, try another time" ✅

KEY CHANGES:
-----------
await client.query('SET TRANSACTION ISOLATION LEVEL SERIALIZABLE');

This tells PostgreSQL: "If two transactions conflict, rollback one automatically"
