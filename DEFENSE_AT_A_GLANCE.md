## 🔍 CONFLICT DETECTION LOGIC

```
BOOKING REQUEST
    ↓
CHECK PATIENT AVAILABILITY
├─ Query appointments for patient at requested time
├─ Query composite_bookings for patient
└─ If conflict → REJECT
    ↓
CHECK DENTIST AVAILABILITY
├─ Query appointments for dentist
├─ Query composite_booking_appointments
└─ If conflict → REJECT
    ↓
CHECK SERVICE DURATION
├─ Verify service duration fits in slot
└─ If doesn't fit → REJECT
    ↓
ALL CHECKS PASS → ALLOW BOOKING ✅
```
